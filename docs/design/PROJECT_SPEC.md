# Video2Skill 项目规范

## 1. 目标

Video2Skill 是一个面向教学视频的本地优先知识处理工具。
首版目标不是“做大而全的平台”，而是固定一条可重复的主链路：

`source -> inspect -> task -> evidence -> knowledge -> audit -> output`

## 2. 首版范围

### 2.1 允许的输入

- 本地视频文件
- B 站公开视频链接

### 2.2 首版交互

- CLI
- 本地网页入口

### 2.3 首版输出

- Markdown 技术文档

### 2.4 明确不做

- 桌面壳
- 商业模型 API
- 自动绕过访问控制
- 无证据生成知识
- 直接执行未审计的 Skill 命令

## 3. 核心对象

### 3.1 TaskManifest

任务清单必须记录：

- 任务 ID
- 来源描述
- 当前状态
- 已完成步骤
- 工件列表
- 告警列表
- 输出目录

### 3.2 EvidenceUnit

证据单元必须包含：

- 时间范围
- 来源列表
- 原始片段
- 置信度

### 3.3 KnowledgeUnit

知识单元只允许从证据中抽取，不允许凭空补写事实。
命令、路径、版本号等高风险内容必须能追溯到证据引用。

### 3.4 AuditReport

审计结果必须说明：

- 覆盖率
- 冲突数
- 待确认项
- 最终决策

## 4. 模块边界

- `launcher`：启动、自检、环境准备
- `apps`：CLI 与本地网页入口
- `adapters`：来源适配
- `core`：任务、证据、知识、审计
- `tools`：FFmpeg、ASR、OCR、渲染器
- `docs` / `configs` / `tests`：规范、默认值、验证

模块之间只允许依赖契约层，不允许直接穿透内部实现。

## 5. 质量门槛

- 本地文件主流程必须可独立验证
- B 站适配必须可关闭、可回退
- 无字幕时应有 ASR 回退路径
- 没有证据的内容只能进入“待确认”区
- 每个 feature 必须可单独测试、单独回退

## 6. 阶段规划

- Phase A：规范冻结与骨架整理
- Phase B：来源检查与字幕/ASR 闭环
- Phase C：证据融合与 OCR 扩展
- Phase D：知识抽取与 Markdown 输出
